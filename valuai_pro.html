Architecture
How ValuAI Works
📍
Input & Geocoding
Address, property type, size, age entered by user. System geocodes to micro-market, validates inputs, runs fraud/anomaly checks.
🗄
Data Acquisition
Karnataka guidance values, OpenStreetMap amenity layers, listing density proxies, transport datasets (BMTC/metro), neighbourhood typology.
⚙
Feature Engineering
Location intelligence, property characteristics, legal attributes, income signals, and anomaly flags computed into rich feature set.
🧠
Dual-Engine Scoring
Valuation engine computes market & distress value bands. Liquidity engine produces resale index, liquidation timeline, confidence score.
✦
Explainability Layer
Driver ranking, risk flag decomposition, confidence reasoning, and policy recommendation surfaced as underwriter-readable outputs.
🛡
Governance & Audit
Immutable audit logs, model versioning, human override flags, human review thresholds for low-confidence or high-ticket cases.
Market Value Formula
Market Value = Base Guidance Value × Location Multiplier × Property Adjustment × Condition/Age Adjustment × Marketability Adjustment
Layer 1: Rules-based floor & cap (guidance value + size sanity)
Layer 2: Gradient boosting on engineered features → fair market band
Layer 3: Quantile regression / conformal prediction → lower & upper bounds
Distress Value
Distress Value = Market Value × (1 − Liquidity Discount)
Prime apartment (clear title): 10–18% · Mid-market: 15–25% · Old/niche/legal issues: 25–40%
Resale Potential Index
RPI = 0.30·L + 0.20·F + 0.20·J + 0.10·A + 0.10·R + 0.10·M
L=Location · F=Fungibility · J=Legal clarity · A=Age · R=Rental attractiveness · M=Market balance
